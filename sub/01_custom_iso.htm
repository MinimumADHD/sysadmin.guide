Panoramica & Strumenti Necessari

Tutto ciò che serve prima di iniziare

Creare una ISO Windows custom significa preparare un'immagine di sistema che, una volta distribuita, presenta già un ambiente configurato e pronto all'uso. Questo processo si articola in tre macro-fasi: preparazione in VM, generalizzazione con Sysprep e post-processing dell'immagine.

🖥️

VMware Workstation

Ambiente di lavoro isolato per installare e configurare Windows senza toccare il sistema host. Versione 16+ consigliata.

🗑️

BCUninstaller (Bulk Crap)

Disinstallatore di massa per rimuovere bloatware, app UWP e residui di registro in modo chirurgico.

✂️

NTLite

Editor avanzato per immagini WIM/ESD. Permette di rimuovere componenti, integrare driver e aggiornamenti.

🔧

DISMTools

GUI per DISM. Gestione montaggio immagini, integrazione patch, conversione ESD↔WIM.

📦

Sysprep

Tool Microsoft integrato in Windows per generalizzare l'immagine e renderla pronta per la distribuzione.

💿

oscdimg / AnyBurn

Tool per creare l'ISO finale avviabile da un disco montato o da una cartella di files estratti.

ISO Windows originale

Scarica sempre da fonti ufficiali. Usa lo strumento di creazione multimediale Microsoft oppure scarica direttamente tramite PowerShell con il modulo Get-WindowsImage, oppure da https://www.microsoft.com/software-download/windows11.

⚠️
Importante: Licenze Assicurati di avere le licenze adeguate per distribuire Windows customizzato. Le OEM license e le licenze Volume (KMS/MAK) hanno condizioni differenti. Non redistribuire immagini attivate senza autorizzazione.

Setup Macchina Virtuale VMware

Configurazione ottimale della VM di lavoro

La VM è il nostro "laboratorio". Deve essere abbastanza potente da eseguire installazioni software e abbastanza isolata da non contaminare il sistema host.

💡
Strategia Snapshot Crea uno snapshot dopo ogni fase critica: post-installazione OS pulita, post-installazione software, pre-sysprep. Questo ti permette di tornare indietro senza ricominciare da capo.

Installazione Windows nella VM

  1. Crea una nuova VM in VMware → Custom (Advanced) → seleziona hardware compatibility dell'ultima versione disponibile.
  2. Monta la ISO Windows originale come CD/DVD drive della VM.
  3. Avvia la VM e procedi con l'installazione standard. Scegli Installazione personalizzata (non aggiornamento).
  4. Durante la configurazione OOBE, non connettere a Internet (premi Shift+F10 e digita oobe\bypassnro per aggirare l'obbligo di account Microsoft su Windows 11).
  5. Crea un account locale temporaneo (es. Administrator o SetupAdmin). Questo account verrà rimosso o generalizzato con Sysprep.
  6. Installa i VMware Tools immediatamente dopo il primo avvio.
  7. Snapshot #1: "OS pulito - pre configurazione".

Creazione Utenti e Profili

Utenti locali, gruppi e configurazione profili predefiniti

Esistono due approcci per creare utenti custom: manualmente tramite GUI/CMD/PS nella VM, oppure tramite un file Unattend.xml che configura tutto durante la fase di Sysprep/OOBE. Il secondo approccio è quello professionale.

Metodo 1 — PowerShell (Diretto nella VM)

POWERSHELL 
# Crea un nuovo utente locale
$Password = ConvertTo-SecureString "P@ssw0rd2024!" -AsPlainText -Force
New-LocalUser -Name "Utente.Standard" `
              -Password $Password `
              -FullName "Utente Standard" `
              -Description "Account utente predefinito azienda" `
              -PasswordNeverExpires

# Aggiunge l'utente al gruppo Users (non admin)
Add-LocalGroupMember -Group "Users" -Member "Utente.Standard"

# Crea un utente Admin separato
$AdminPass = ConvertTo-SecureString "Adm1n@Secure!" -AsPlainText -Force
New-LocalUser -Name "LocalAdmin" `
              -Password $AdminPass `
              -FullName "Local Administrator" `
              -PasswordNeverExpires
Add-LocalGroupMember -Group "Administrators" -Member "LocalAdmin"

# Disabilita l'account Administrator built-in (sicurezza)
Disable-LocalUser -Name "Administrator"

# Verifica utenti creati
Get-LocalUser | Format-Table Name, Enabled, PasswordLastSet

Il file unattend.xml va posizionato in C:\Windows\Panther\ prima di eseguire Sysprep. Viene letto automaticamente da Windows Setup durante le fasi specialize e oobeSystem.

Creazione Cartelle Default

Struttura directory standardizzata per tutti gli utenti

Le cartelle vanno create nel profilo Default User (C:\Users\Default\) in modo che ogni nuovo utente le erediti automaticamente al primo accesso.

Script PowerShell — Creazione struttura cartelle

POWERSHELL · Setup-DefaultFolders.ps1 
# ============================================
# Setup Cartelle Profilo Default
# Eseguire come SYSTEM o Administrator
# ============================================

$DefaultProfile = "C:\Users\Default"

# Struttura cartelle da creare
$Folders = @(
    "Desktop\Strumenti IT",
    "Desktop\Documenti Aziendali",
    "Documents\Progetti",
    "Documents\Modelli",
    "Documents\Archivio",
    "Downloads\Software",
    "Pictures\Loghi Aziendali",
    "AppData\Roaming\AziendaApp"
)

foreach ($folder in $Folders) {
    $FullPath = Join-Path $DefaultProfile $folder
    if (-not (Test-Path $FullPath)) {
        New-Item -ItemType Directory -Path $FullPath -Force | Out-Null
        Write-Host "[✓] Creata: $FullPath" -ForegroundColor Green
    } else {
        Write-Host "[~] Già esistente: $FullPath" -ForegroundColor Yellow
    }
}

# Copia file template nelle cartelle (es. modelli Office)
$TemplatesSource = "C:\Setup\Templates"
if (Test-Path $TemplatesSource) {
    Copy-Item -Path "$TemplatesSource\*" `
              -Destination "$DefaultProfile\Documents\Modelli" `
              -Recurse -Force
    Write-Host "[✓] Template copiati" -ForegroundColor Green
}

# Imposta sfondo Desktop (da file nel profilo Default)
$WallpaperSrc = "C:\Setup\wallpaper.jpg"
$WallpaperDst = "C:\Windows\Web\Wallpaper\Company\wallpaper.jpg"
if (Test-Path $WallpaperSrc) {
    New-Item -ItemType Directory -Path "C:\Windows\Web\Wallpaper\Company" -Force | Out-Null
    Copy-Item $WallpaperSrc $WallpaperDst -Force
    Write-Host "[✓] Wallpaper copiato" -ForegroundColor Green
}

Write-Host "`n[DONE] Struttura cartelle completata." -ForegroundColor Cyan

Configurazione Registry per cartelle speciali

POWERSHELL · Reindirizza cartelle tramite Registry 
# Carica hive del profilo Default
reg load HKLM\DefaultUser C:\Users\Default\NTUSER.DAT

# Imposta sfondo desktop per il profilo Default
Set-ItemProperty -Path "HKLM:\DefaultUser\Control Panel\Desktop" `
    -Name "Wallpaper" `
    -Value "C:\Windows\Web\Wallpaper\Company\wallpaper.jpg"

# Disabilita la schermata di blocco (opzionale)
Set-ItemProperty -Path "HKLM:\DefaultUser\Software\Policies\Microsoft\Windows\Personalization" `
    -Name "NoLockScreen" `
    -Value 1 -Type DWord -Force

# Scollega il profilo Default (OBBLIGATORIO dopo reg load)
reg unload HKLM\DefaultUser

Preinstallazione Software

Installazione silenziosa e automatizzata di applicazioni

L'installazione del software deve essere silenziosa (senza interazione utente) per essere scriptabile. La maggior parte dei setup supporta parametri silent. L'approccio migliore è creare uno script master che esegue tutto in sequenza.

Parametri silent comuni per installer

Tipo InstallerParametri SilentNote
NSIS (.exe)/SCase-sensitive, maiuscolo
Inno Setup (.exe)/SILENT o /VERYSILENT/SUPPRESSMSGBOXES per nessun popup
MSI/quiet /norestartUsare msiexec /i setup.msi /quiet
InstallShield/s /v"/qn"Variabile secondo versione
7-Zip SFX-yRisponde "sì" a tutto
WinGet--silent --accept-*Metodo moderno consigliato

Script master di installazione

POWERSHELL · Install-Software.ps1 
# ============================================
# Master Script - Installazione Software
# Posiziona questo script e i setup in C:\Setup\SW\
# Eseguire come Administrator
# ============================================

$LogFile = "C:\Setup\install_log.txt"
$SWPath  = "C:\Setup\SW"

function Write-Log { param($msg) Add-Content $LogFile "[$(Get-Date -f 'HH:mm:ss')] $msg" }
function Install-Silent {
    param($Name, $Exe, $Args)
    Write-Host "  → Installando $Name..." -NoNewline
    $proc = Start-Process -FilePath $Exe -ArgumentList $Args `
                          -Wait -PassThru -WindowStyle Hidden
    if ($proc.ExitCode -eq 0) {
        Write-Host " [OK]" -ForegroundColor Green
        Write-Log "[OK] $Name"
    } else {
        Write-Host " [ERR: $($proc.ExitCode)]" -ForegroundColor Red
        Write-Log "[FAIL] $Name - ExitCode: $($proc.ExitCode)"
    }
}

Write-Host "`n========== INSTALLAZIONE SOFTWARE ==========" -ForegroundColor Cyan

# ── Produttività ──────────────────────────────
Install-Silent "7-Zip"          "$SWPath\7z2301-x64.exe"   "/S"
Install-Silent "VLC Media Player" "$SWPath\vlc-3.0.20.exe" "/L=1040 /S"
Install-Silent "Notepad++"       "$SWPath\npp.8.5.7.exe"   "/S"
Install-Silent "Adobe Reader DC"  "$SWPath\AcroRdr.exe"     "/sAll /rs /msi EULA_ACCEPT=YES"

# ── Browser ───────────────────────────────────
Install-Silent "Google Chrome"   "$SWPath\ChromeSetup.exe"  "/silent /install"

# ── Runtime & Framework ───────────────────────
Install-Silent ".NET 8 Runtime"  "$SWPath\dotnet-8-x64.exe" "/quiet /norestart"
Install-Silent "VC++ 2022"       "$SWPath\VC_redist.x64.exe""/quiet /norestart"

# ── Tramite WinGet (se disponibile) ───────────
if (Get-Command winget -ErrorAction SilentlyContinue) {
    Write-Host "`n[WinGet] Installazione pacchetti aggiuntivi..."
    $packages = @(
        "Git.Git",
        "Microsoft.WindowsTerminal",
        "VideoLAN.VLC"
    )
    foreach ($pkg in $packages) {
        winget install --id $pkg --silent --accept-package-agreements --accept-source-agreements
    }
}

# ── MSI tramite msiexec ───────────────────────
msiexec /i "$SWPath\app.msi" /quiet /norestart /log "C:\Setup\msi_log.txt"

Write-Host "`n[DONE] Installazione completata. Log: $LogFile" -ForegroundColor Cyan

Rimozione Software con BCUninstaller

Eliminazione chirurgica di bloatware, app UWP e residui

Bulk Crap Uninstaller (BCUninstaller) è lo strumento più efficace per rimuovere software da Windows in modo pulito: rileva app tradizionali, Store/UWP, Chocolatey packages e i relativi residui di registro.

Configurazione BCUninstaller per uso in ISO

  1. Scarica BCUninstaller Portable da https://www.bcuninstaller.com e copialo in C:\Setup\Tools\BCU\.
  2. Avvia BCUninstaller.exe dalla cartella portable — non richiede installazione.
  3. Vai su Options → Settings e abilita: "Show Microsoft updates", "Show protected components" e "Show Windows Store apps".
  4. Attendi il caricamento completo dell'elenco (può richiedere 30–60 secondi).
  5. Seleziona le app da rimuovere (vedi lista consigliata sotto).
  6. Usa Tools → Create bulk uninstall task per creare un task riutilizzabile.
  7. Durante la disinstallazione attiva: Quiet uninstall e Delete leftovers after uninstall.

Registry & Group Policy

Ottimizzazioni di sistema, privacy e impostazioni aziendali

Tweaks di sistema via Registry

POWERSHELL · System-Tweaks.ps1 
# ============================================
# Ottimizzazioni Registry per immagine custom
# ============================================

# ── PRIVACY & TELEMETRIA ─────────────────────
# Disabilita telemetria (0=Security, 1=Basic)
Set-ItemProperty "HKLM:\SOFTWARE\Policies\Microsoft\Windows\DataCollection" `
    "AllowTelemetry" 0 -Type DWord -Force

# Disabilita advertising ID
Set-ItemProperty "HKLM:\SOFTWARE\Policies\Microsoft\Windows\AdvertisingInfo" `
    "DisabledByGroupPolicy" 1 -Type DWord -Force

# ── PERFORMANCE ──────────────────────────────
# Disabilita search index per disco SSD (opzionale)
Set-ItemProperty "HKLM:\SYSTEM\CurrentControlSet\Services\WSearch" `
    "Start" 4 -Type DWord  # 4 = Disabled

# Imposta Power Plan su "High Performance"
powercfg /setactive 8c5e7fda-e8bf-4a96-9a85-a6e23a8c635c

# ── UI & EXPLORER ────────────────────────────
# Mostra estensioni file nel Explorer
Set-ItemProperty "HKCU:\Software\Microsoft\Windows\CurrentVersion\Explorer\Advanced" `
    "HideFileExt" 0 -Type DWord

# Mostra file nascosti
Set-ItemProperty "HKCU:\Software\Microsoft\Windows\CurrentVersion\Explorer\Advanced" `
    "Hidden" 1 -Type DWord

# Apri Explorer su "Questo PC" invece di "Accesso rapido"
Set-ItemProperty "HKCU:\Software\Microsoft\Windows\CurrentVersion\Explorer\Advanced" `
    "LaunchTo" 1 -Type DWord

# Disabilita Widgets (Windows 11)
Set-ItemProperty "HKLM:\SOFTWARE\Policies\Microsoft\Dsh" `
    "AllowNewsAndInterests" 0 -Type DWord -Force

# Disabilita Chat (Teams Personal taskbar - Win11)
Set-ItemProperty "HKLM:\SOFTWARE\Policies\Microsoft\Windows\Windows Chat" `
    "ChatIcon" 3 -Type DWord -Force

# ── WINDOWS UPDATE ────────────────────────────
# Pausa aggiornamenti automatici (business image)
Set-ItemProperty "HKLM:\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate\AU" `
    "NoAutoUpdate" 1 -Type DWord -Force

Write-Host "[DONE] Registry tweaks applicati." -ForegroundColor Green

Sysprep & Generalizzazione

Passaggio critico prima dell'acquisizione dell'immagine

Sysprep generalizza l'immagine rimuovendo identificatori univoci (SID, nome computer, licenza hardware) e la rende deployabile su hardware diverso. È un passaggio obbligatorio e irreversibile: fai uno snapshot prima.

🛑
SNAPSHOT prima di Sysprep Sysprep è irreversibile sulla VM. Crea Snapshot #3: "Pre-Sysprep FINAL" prima di procedere. Questo ti permette di tornare indietro se l'unattend.xml ha errori.

Preparazione pre-Sysprep

POWERSHELL · Pre-Sysprep Cleanup 
# Pulizia prima di Sysprep

# Svuota cestino
Clear-RecycleBin -Force -ErrorAction SilentlyContinue

# Pulisci Windows Update cache
Stop-Service wuauserv -Force
Remove-Item "C:\Windows\SoftwareDistribution\Download\*" -Recurse -Force -ErrorAction SilentlyContinue
Start-Service wuauserv

# Pulisci temp files
Remove-Item "C:\Windows\Temp\*" -Recurse -Force -ErrorAction SilentlyContinue
Remove-Item "$env:TEMP\*"        -Recurse -Force -ErrorAction SilentlyContinue

# Rimuovi cartella C:\Setup (non serve nell'immagine finale)
# Remove-Item "C:\Setup" -Recurse -Force  # Decommentare quando pronto

# Pulizia Disk (silenzioso)
cleanmgr /sagerun:1

# Deframmenta (solo HDD, non SSD)
# defrag C: /U /V

Write-Host "[DONE] Sistema pronto per Sysprep." -ForegroundColor Green

Esecuzione Sysprep

CMD · Sysprep con Unattend.xml 
REM Posiziona l'unattend.xml in C:\Windows\System32\Sysprep\
REM poi esegui dal CMD con privilegi Administrator:

cd C:\Windows\System32\Sysprep

REM Generalizza + Shutdown (per acquisire immagine)
sysprep.exe /oobe /generalize /shutdown /unattend:unattend.xml

REM Oppure /reboot per riavviare invece di spegnere
REM /oobe = mostra OOBE al primo avvio (ma skippa se configurato in XML)
REM /generalize = rimuove SID e info hardware-specifiche
REM /shutdown = spegne la VM dopo la generalizzazione
💡
Cosa fa /generalize Rimuove il Security ID (SID) univoco, le informazioni di attivazione hardware-bound, i log di evento, i certificati di sistema e i punti di ripristino. La VM risultante non è più avviabile normalmente, ma è pronta per essere catturata come immagine.

Esportazione ISO

Acquisizione immagine WIM e creazione ISO avviabile

Dopo che Sysprep ha spento la VM, il disco virtuale contiene l'immagine generalizzata. Dobbiamo: 1) catturare la partizione Windows in un file WIM, 2) iniettarlo nella struttura della ISO originale, 3) creare la ISO avviabile.

Metodo A — Acquisizione tramite Windows PE + DISM

Avvia la VM con una ISO di Windows PE o usa l'ambiente WinRE. Da lì monta il disco della VM e acquisici l'immagine.

CMD · Acquisizione immagine WIM con DISM (da WinPE) 
REM Da WinPE, identifica la lettera del drive Windows:
diskpart
  list disk
  select disk 0
  list volume
  exit

REM Cattura l'immagine del volume Windows (es. D:) in WIM
dism /Capture-Image `
     /ImageFile:E:\custom_windows.wim `
     /CaptureDir:D:\ `
     /Name:"Windows 11 Pro Custom" `
     /Description:"Immagine custom aziendale v1.0" `
     /Compress:maximum `
     /CheckIntegrity

REM Nota: E: deve essere un disco separato o USB dove salvare il WIM

Metodo B — Acquisizione dall'host VMware (consigliato)

Dopo lo shutdown della VM da Sysprep, monta il VMDK direttamente sull'host usando VMware Disk Mount Utility o monta il VMDK come disco aggiuntivo in un'altra VM con WinPE.

CMD · Monta VMDK su host Windows con VMware DiskMount 
REM VMware DiskMount Utility (vmware-mount.exe)
REM Disponibile con VMware Workstation Pro

NTLite — Editing Avanzato Immagine

Rimozione componenti, integrazione driver e aggiornamenti

NTLite ti permette di modificare l'immagine WIM offline (senza avviarla): rimuovere feature Windows, integrare driver, applicare aggiornamenti cumulativi e configurare impostazioni avanzate. La versione Free copre le funzioni di base; la versione a pagamento sblocca automazione e task XML.

Workflow NTLite

  1. Aggiungi immagine: File → Add → Image file → seleziona install.wim dalla ISO estratta.
  2. Carica l'edizione: Doppio click sull'edizione desiderata (es. "Windows 11 Pro") per montarla.
  3. Components: Naviga l'albero delle componenti → rimuovi quelle non necessarie (Internet Explorer, Media Player legacy, Tablet components, etc.).
  4. Updates: Integra aggiornamenti cumulativi in formato .msu o .cab scaricati da Microsoft Update Catalog.
  5. Drivers: Integra driver (cartella con .inf) per hardware specifico.
  6. Unattend: Configura il file di risposta direttamente da NTLite (GUI per unattend.xml).
  7. Apply: Avvia l'elaborazione → NTLite salva il WIM modificato.

Componenti consigliati da rimuovere con NTLite

ComponentePercorso in NTLiteNote
Internet ExplorerComponents → BrowsersDeprecato, non serve
Windows Media Player (legacy)Components → MultimediaUWP Media è il sostituto
WordPadComponents → Desktop AppsRimosso da Win11 23H2+
Windows Fax and ScanComponents → CommunicationRaramente usato
XPS ServicesComponents → PrintingSe si usa solo PDF
OneDrive (built-in)Components → OneDriveSe non in uso aziendale
Microsoft StoreComponents → Windows Store⚠️ Solo se sicuri, può rompere UWP

DISMTools — Gestione Immagine

GUI per DISM: montaggio, patch, conversione ESD/WIM

DISMTools è una GUI per il tool DISM integrato in Windows, che semplifica operazioni complesse come montaggio immagini, integrazione aggiornamenti, verifica integrità e conversione tra formati.

Operazioni principali in DISMTools

  1. Mount Image: Monta un WIM/ESD in una cartella per modificarlo offline (equivale a dism /Mount-Image).
  2. Add Packages: Integra update .msu o .cab nell'immagine montata.
  3. Add Drivers: Inietta driver .inf nell'immagine.
  4. Cleanup Image: Riduce le dimensioni rimuovendo componenti superseded.
  5. Unmount Image: Smonta salvando le modifiche (commit) o scartandole.
  6. Export Image: Esporta in WIM compresso massimo per ridurre dimensioni.
  7. Convert ESD→WIM: Necessario se la ISO usa install.esd invece di install.wim.

Comandi DISM equivalenti da riga di comando

CMD / PowerShell · Operazioni DISM principali 
REM ── MONTA IMMAGINE ────────────────────────────
dism /Mount-Image /ImageFile:D:\install.wim `
     /index:1 /MountDir:D:\Mount /ReadWrite

REM ── VISUALIZZA INFORMAZIONI ─────────────────
dism /Get-ImageInfo /ImageFile:D:\install.wim

REM ── INTEGRA AGGIORNAMENTO ───────────────────
dism /Image:D:\Mount /Add-Package `
     /PackagePath:D:\Updates\windows11-kb5034123.msu

REM ── AGGIUNGI DRIVER ─────────────────────────
dism /Image:D:\Mount /Add-Driver `
     /Driver:D:\Drivers\ /Recurse

REM ── CLEANUP (riduce size) ───────────────────
dism /Image:D:\Mount /Cleanup-Image /StartComponentCleanup /ResetBase

REM ── SMONTA E SALVA ──────────────────────────
dism /Unmount-Image /MountDir:D:\Mount /Commit

REM ── ESPORTA (compressione massima) ──────────
dism /Export-Image /SourceImageFile:D:\install.wim `
     /SourceIndex:1 /DestinationImageFile:D:\install_compressed.wim `
     /Compress:max

REM ── CONVERTI ESD → WIM ──────────────────────
dism /Export-Image /SourceImageFile:D:\install.esd `
     /SourceIndex:1 /DestinationImageFile:D:\install.wim `
     /Compress:max /CheckIntegrity

REM ── VERIFICA INTEGRITÀ ──────────────────────
dism /Check-ImageHealth /ImageFile:D:\install.wim

Checklist Finale & Best Practice

Tutto ciò che non deve mancare prima della distribuzione

Checklist pre-distribuzione

  1. VM testata: La ISO è stata installata in una VM di test pulita senza errori.
  2. Utenti verificati: Tutti gli account creati correttamente con permessi giusti.
  3. Software verificato: Tutte le applicazioni funzionano al primo avvio, shortcut corretti.
  4. Bloatware rimosso: Nessuna app UWP indesiderata presente nel menu Start.
  5. Cartelle create: Struttura directory presente in nuovo profilo utente.
  6. Sfondo/tema: Wallpaper aziendale applicato correttamente.
  7. Dimensione ISO: Verificata e ottimizzata (WIM esportato con compressione massima).
  8. UEFI/Legacy boot: ISO avviabile sia in UEFI che Legacy BIOS se necessario.
  9. Unattend.xml validato: Usa Windows SIM (parte di ADK) per validare il file XML.
  10. Log esaminati: Controlla C:\Windows\Panther\setupact.log e setuperr.log dopo test.

Script setup completo (master launcher)

POWERSHELL · Master-Setup.ps1 — Lancia tutto in sequenza 
# ============================================
# MASTER SETUP SCRIPT
# Da eseguire come Administrator nella VM
# dopo installazione Windows, prima di Sysprep
# ============================================

$Steps = @(
    @{"Name"="Cartelle Default";   "Script"="C:\Setup\Setup-DefaultFolders.ps1"},
    @{"Name"="Rimozione Bloatware"; "Script"="C:\Setup\Remove-Bloatware.ps1"},
    @{"Name"="Installazione SW";    "Script"="C:\Setup\Install-Software.ps1"},
    @{"Name"="Registry Tweaks";    "Script"="C:\Setup\System-Tweaks.ps1"}
)

foreach ($step in $Steps) {
    Write-Host "`n━━━ $($step.Name) ━━━" -ForegroundColor Cyan
    if (Test-Path $step.Script) {
        & $step.Script
    } else {
        Write-Warning "Script non trovato: $($step.Script)"
    }
}

# Snapshot consigliato prima di procedere con Sysprep
Write-Host "`n[!] CREA SNAPSHOT ORA prima di Sysprep!" -ForegroundColor Yellow
Read-Host "Premi INVIO quando pronto per Sysprep..."

# Pulizia finale
Remove-Item "C:\Windows\Temp\*"   -Recurse -Force -ErrorAction SilentlyContinue
Remove-Item "$env:TEMP\*"          -Recurse -Force -ErrorAction SilentlyContinue

# Lancia Sysprep
Start-Process "C:\Windows\System32\Sysprep\sysprep.exe" `
    -ArgumentList "/oobe /generalize /shutdown /unattend:C:\Windows\System32\Sysprep\unattend.xml" `
    -Wait
