Capitolo 1

I Livelli del Sistemista

Il lavoro del sistemista IT è strutturato in livelli che definiscono la complessità delle attività, l'autonomia operativa e le competenze richieste. Comprendere questa gerarchia è fondamentale per orientarsi nella carriera e nel lavoro quotidiano.

L1 Primo Livello

Help Desk & Supporto Operativo

Il sistemista di primo livello è il primo punto di contatto con gli utenti finali. Gestisce ticket, risolve problemi comuni, esegue operazioni di routine e scala i problemi complessi ai livelli superiori. Lavora principalmente seguendo procedure documentate (SOP).

Gestione Ticket Reset Password Assistenza PC Office 365 Base Printer Setup VPN Client Antivirus Onboarding Utenti
L2 Secondo Livello

Sistemista Server & Infrastruttura

Il sistemista di secondo livello gestisce server, virtualizzazione, Active Directory, reti e servizi aziendali. Ha autonomia operativa elevata, gestisce incidenti complessi, pianifica manutenzioni e mantiene l'infrastruttura. È il backbone tecnico dell'azienda.

Windows Server Active Directory VMware/Hyper-V Exchange/M365 Backup & DR Firewall DNS/DHCP Scripting
L3 Terzo Livello

Architetto / Senior Infrastructure Engineer

Il sistemista di terzo livello è un architetto che progetta soluzioni, gestisce progetti complessi, definisce standard e policy aziendali. Spesso lavora con vendor, consulenti e management. Possiede specializzazioni deep in una o più aree (sicurezza, cloud, networking, storage).

Cloud Architecture Azure / AWS Zero Trust Disaster Recovery vSAN / SDS SIEM / SOC IAM / PKI DevOps / IaC

Componenti Fondamentali

🏢

Domain Controller (DC)

Server Windows con ruolo AD DS. Gestisce autenticazione, policy di gruppo (GPO), DNS interno e DHCP. In ambienti enterprise si raccomanda almeno 2 DC per ridondanza (DC1 + DC2).

🖥

Server di Virtualizzazione

Host VMware ESXi o Hyper-V che eseguono le macchine virtuali. In cluster permettono alta disponibilità (HA) e migrazione live (vMotion/Live Migration) senza downtime.

📁

File Server / NAS

Storage condiviso aziendale. Può essere Windows Server con DFS, un NAS Synology/QNAP, o storage enterprise (NetApp, EMC). Gestisce permessi NTFS e condivisioni SMB/NFS.

🛡

Firewall & UTM

Prima linea di difesa perimetrale. Gestisce regole di accesso, NAT, VPN site-to-site e client, IPS/IDS, web filtering. Soluzioni comuni: Fortinet, pfSense, Sophos, Palo Alto.

🔀

Switch Managed L2/L3

Gestione VLAN, STP/RSTP, aggregazione link (LACP), QoS per voce/video. Gli switch L3 gestiscono il routing tra VLAN evitando di sovraccaricare il firewall.

☁

Cloud Ibrido

Microsoft Azure AD Connect sincronizza le identità on-premise verso il cloud. Azure AD permette SSO, MFA e accesso sicuro a M365, Teams, SharePoint e app SaaS esterne.

Capitolo 3

Networking & Diagnostica di Rete

La diagnostica di rete è tra le attività più frequenti sia al 1° che al 2° livello. Un sistemista deve saper rilevare problemi di connettività, DNS, routing e latenza.

Comandi di Diagnostica Base — Windows vs Linux

📡 Test Connettività (Ping)

Windows (CMD / PowerShell)
# Ping standard
ping 8.8.8.8
ping google.com -t        # continuo
ping 192.168.1.1 -n 10   # 10 pacchetti

# Ping con IPv6
ping -6 google.com

# PowerShell
Test-Connection -ComputerName google.com -Count 4
Test-NetConnection -ComputerName server01 -Port 443
Linux (Bash)
# Ping standard
ping 8.8.8.8
ping -c 4 google.com     # 4 pacchetti
ping -i 0.5 192.168.1.1 # intervallo 0.5s

# Ping con IPv6
ping6 google.com

# Ping con verbose/dettagli
ping -v -c 5 192.168.1.1

🔍 Traceroute — Percorso dei Pacchetti

Windows
tracert google.com
tracert -d 8.8.8.8    # no DNS resolve
# PowerShell
Test-NetConnection -ComputerName google.com -TraceRoute
Linux
traceroute google.com
traceroute -n 8.8.8.8   # no DNS resolve
mtr google.com          # realtime (installare mtr)
tracepath google.com    # no root required

🌐 DNS — Risoluzione Nomi

Windows
nslookup google.com
nslookup google.com 8.8.8.8  # DNS specifico
nslookup -type=MX gmail.com   # record MX
ipconfig /flushdns             # svuota cache DNS
ipconfig /displaydns           # mostra cache
# PowerShell
Resolve-DnsName google.com
Resolve-DnsName google.com -Type MX
Linux
nslookup google.com
dig google.com              # dettagliato
dig @8.8.8.8 google.com    # DNS specifico
dig google.com MX          # record MX
host google.com            # semplice
# Svuota cache systemd-resolved
sudo systemd-resolve --flush-caches

🔌 Porte e Connessioni Attive

Windows
netstat -ano              # connessioni + PID
netstat -ano | findstr :80 # filtra porta
# PowerShell
Get-NetTCPConnection
Get-NetTCPConnection -State Listen
Test-NetConnection -ComputerName server01 -Port 3389
Linux
netstat -tulnp           # porte in ascolto
ss -tulnp               # più moderno
ss -tulnp | grep :80    # filtra porta
lsof -i :443            # processo su porta
nmap -sV 192.168.1.100  # scan porte host

⚙ Configurazione Interfacce di Rete

Windows
ipconfig /all            # info complete
ipconfig /release        # rilascia IP DHCP
ipconfig /renew          # rinnova IP DHCP
# PowerShell
Get-NetIPAddress
Get-NetAdapter
New-NetIPAddress -InterfaceAlias "Ethernet" -IPAddress 10.0.0.10 -PrefixLength 24 -DefaultGateway 10.0.0.1
Linux
ip addr show           # info interfacce
ip route show          # tabella routing
ip link set eth0 up    # abilita interfaccia
# Assegna IP temporaneo
ip addr add 10.0.0.10/24 dev eth0
# Rinnova DHCP
sudo dhclient -r eth0 && sudo dhclient eth0
💡 Pro Tip — 2° Livello Quando un utente segnala "internet non funziona", il sistemista segue il percorso logico: ping al gateway → ping al DNS → ping a 8.8.8.8 → nslookup → traceroute. Questo permette di isolare immediatamente il punto di guasto (rete locale, DNS, WAN o destinazione).
Capitolo 4

Windows Server — Gestione e Amministrazione

Windows Server è il sistema operativo server più diffuso nelle PMI italiane. Il sistemista di 2° livello deve padroneggiarne l'amministrazione quotidiana.

Ruoli Server più Comuni

🔐

AD DS — Active Directory Domain Services

Cuore dell'identità aziendale. Gestisce utenti, computer, gruppi, GPO e trust. Richiede almeno 2 DC per alta disponibilità.

🌐

DNS & DHCP

DNS risolve i nomi interni (server01.azienda.local). DHCP assegna IP automaticamente ai client. Spesso co-installati sul DC.

📂

File & Storage Services

Condivisioni SMB, gestione quote, shadow copy, DFS (Distributed File System) per namespace distribuiti e replica cartelle.

🖨

Print Services

Gestione centralizzata delle stampanti di rete, driver deployment via GPO, print queue monitoring e troubleshooting code di stampa.

🔒

IIS — Web Server

Hosting applicazioni web .NET, gestione certificati SSL/TLS, application pool, binding siti, log accessi e configurazione sicurezza.

🔄

WSUS — Windows Update

Gestione centralizzata degli aggiornamenti Windows per server e workstation. Permette di approvare, testare e distribuire patch in modo controllato.

Comandi PowerShell Essenziali per Windows Server

Gestione Servizi

PowerShell — Gestione Servizi
# Visualizza tutti i servizi
Get-Service

# Avvia, ferma, riavvia un servizio
Start-Service -Name "Spooler"

Restart-Service -Name "W32Time"

# Configura avvio automatico
Set-Service -Name "Spooler" -StartupType Automatic

# Controlla servizio specifico
Get-Service -Name "NTDS" | Select-Object Name, Status, StartType

Monitoraggio Performance

Windows
# CPU e Memoria
Get-Process | Sort-Object CPU -Descending | Select -First 10

# RAM totale e disponibile
Get-CimInstance Win32_OperatingSystem | Select TotalVisibleMemorySize, FreePhysicalMemory

# Top processi per memoria

# CPU e Memoria

# Memoria
free -h

# Processi per CPU/RAM
ps aux --sort=-%cpu | head -10
ps aux --sort=-%mem | head -10

Event Log — Analisi degli Errori

PowerShell — Event Log
# Ultimi 20 errori nel System log
Get-EventLog -LogName System -EntryType Error -Newest 20

# Event Log specifico con filtro
Get-WinEvent -FilterHashtable @{LogName='Application'; Level=2; StartTime=(Get-Date).AddDays(-7)}

# Cerca evento specifico per ID
Get-WinEvent -FilterHashtable @{LogName='Security'; Id=4625} | Select -First 10
# ID 4625 = Login fallito — utile per rilevare attacchi brute force

# Esporta event log
Get-EventLog -LogName System -Newest 100 | Export-Csv C:\Logs\system_errors.csv -NoTypeInformation
⚠ Attenzione Prima di riavviare un servizio critico su un server di produzione, verifica sempre se ci sono utenti connessi (query session o quser) e comunica il downtime. Non agire mai d'impulso su sistemi in produzione.

Gestione Utenti — PowerShell RSAT

PowerShell — Modulo ActiveDirectory
# Importa modulo AD (richiede RSAT)
Import-Module ActiveDirectory

━━━ UTENTI ━━━
# Crea nuovo utente
New-ADUser -Name "Mario Rossi" -SamAccountName "mrossi" -UserPrincipalName "mrossi@azienda.local" -Path "OU=Utenti,DC=azienda,DC=local" -AccountPassword (ConvertTo-SecureString "P@ssw0rd123!" -AsPlainText -Force) -Enabled $true

# Modifica utente
Set-ADUser -Identity "mrossi" -Title "Responsabile IT" -Department "IT" -Office "Milano"

# Reset password
Set-ADAccountPassword -Identity "mrossi" -Reset -NewPassword (ConvertTo-SecureString "NuovaPass123!" -AsPlainText -Force)
Set-ADUser -Identity "mrossi" -ChangePasswordAtLogon $true

# Disabilita / abilita account
Disable-ADAccount -Identity "mrossi"
Enable-ADAccount -Identity "mrossi"

# Sblocca account bloccato
Unlock-ADAccount -Identity "mrossi"

# Cerca utenti inattivi da 90 giorni
Search-ADAccount -AccountInactive -TimeSpan 90 -UsersOnly | Select Name, LastLogonDate

━━━ GRUPPI ━━━
# Aggiungi utente a gruppo
Add-ADGroupMember -Identity "GRP_VPN_Users" -Members "mrossi"

# Rimuovi da gruppo
Remove-ADGroupMember -Identity "GRP_VPN_Users" -Members "mrossi" -Confirm:$false

# Membri di un gruppo
Get-ADGroupMember -Identity "Domain Admins" | Select Name, SamAccountName

# Gruppi di un utente
Get-ADPrincipalGroupMembership -Identity "mrossi" | Select Name

━━━ COMPUTER ━━━
# Lista computer del dominio
Get-ADComputer -Filter * | Select Name, OperatingSystem, LastLogonDate

# Computer inattivi
Get-ADComputer -Filter {LastLogonDate -lt (Get-Date).AddDays(-60)} | Select Name

Group Policy Objects (GPO)

Le GPO sono uno degli strumenti più potenti di Active Directory. Permettono di applicare configurazioni, restrizioni e software a utenti e computer del dominio in modo centralizzato.

🔒

GPO Sicurezza

Policy per password (lunghezza minima, complessità, scadenza), lockout account dopo N tentativi, disabilitazione USB, screen saver con blocco, policy UAC.

🖥

GPO Desktop

Sfondo desktop aziendale, restrizioni al Pannello di Controllo, mappa drive di rete automatica (\\server\share), configurazione proxy, redirezione cartelle.

📦

GPO Software

Distribuzione software MSI/MSIX, whitelist/blacklist applicazioni (AppLocker), configurazione browser (Edge/Chrome policy), certificati trusted.

Comandi GPO Utili

CMD / PowerShell — GPO
# Forza aggiornamento GPO sul client
gpupdate /force

# Report GPO applicato all'utente/computer corrente
gpresult /r
gpresult /h C:\gpreport.html   # report HTML dettagliato
gpresult /user mrossi /h C:\gpreport_mrossi.html

# Forza GPUpdate su PC remoto
Invoke-GPUpdate -Computer "PC001" -Force

# Lista tutte le GPO del dominio
Get-GPO -All | Select DisplayName, GpoStatus, CreationTime
ℹ Best Practice AD Usa il modello Tier 0/1/2 per separare gli account amministrativi: gli account Domain Admin non devono mai essere usati per login interattivi su workstation. Crea account separati: utente normale, admin workstation (L1), admin server (L2), domain admin (solo per AD management).

PowerShell — Microsoft 365

━━━ CONNESSIONE ━━━

━━━ MAILBOX ━━━
# Lista mailbox
Get-Mailbox | Select DisplayName, PrimarySmtpAddress, RecipientTypeDetails

# Info mailbox specifica
Get-Mailbox -Identity "mrossi@azienda.com" | Select *size*, *quota*

# Aggiungi alias email
Set-Mailbox -Identity "mrossi@azienda.com" -EmailAddresses @{Add="mario.rossi@azienda.com"}

# Imposta forward automatico
Set-Mailbox -Identity "mrossi@azienda.com" -DeliverToMailboxAndForward $true -ForwardingAddress "manager@azienda.com"

━━━ MFA ━━━
# Verifica stato MFA utenti

PowerCLI — VMware da PowerShell

VMware PowerCLI (PowerShell)
# Installa e connetti
Install-Module VMware.PowerCLI -Scope CurrentUser
Connect-VIServer -Server vcenter.azienda.local -User admin@vsphere.local

# Lista VM per host
Get-VM | Select Name, PowerState, NumCpu, MemoryGB, VMHost

# Crea snapshot prima di una modifica importante
New-Snapshot -VM "SERVER-WEB01" -Name "Pre-Update-$(Get-Date -Format 'yyyy-MM-dd')" -Description "Snapshot pre-patch"

# vMotion — sposta VM su altro host
Move-VM -VM "SERVER-WEB01" -Destination (Get-VMHost "esxi02.azienda.local")

# Report utilizzo risorse
Get-VM | Get-Stat -Cpu -Memory -MaxSamples 1 | Select Entity, MetricId, Value

# Aumenta RAM a VM (richiede spegnimento con HW memory hot-add off)
Set-VM -VM "SERVER-APP01" -MemoryGB 16 -Confirm:$false

# Lista snapshot vecchi (> 7 giorni) — da rimuovere!
Get-VM | Get-Snapshot | Where {$_.Created -lt (Get-Date).AddDays(-7)} | Select VM, Name, Created, SizeGB
🚨 Regola Aurea sugli Snapshot Gli snapshot VMware NON sono backup! Uno snapshot occupa spazio nel datastore e degrada le performance nel tempo. Rimuovere sempre gli snapshot dopo 24-48 ore. Uno snapshot non rimosso per settimane può causare crash del datastore e perdita di dati.

Configurazione RDP su Windows Server

  1. 1

    Abilita Remote Desktop: Server Manager → Local Server → Remote Desktop → Enable → Allow connections only from computers running Remote Desktop with NLA.

  2. 2

    Configura Firewall: Windows Defender Firewall → Inbound Rules → Remote Desktop (TCP-In) → Enable. Limita gli IP sorgente autorizzati.

  3. 3

    Aggiungi utenti autorizzati: System Properties → Remote tab → Select Users → aggiungi gruppi/utenti specifici. Non usare mai "Everyone".

  4. 4

    Certificato SSL: Per RDP sicuro, associa un certificato SSL al server RDS tramite Group Policy: Computer Configuration → Administrative Templates → Windows Components → Remote Desktop Services.

  5. 5

    Cambia porta RDP (opzionale): Regedit → HKLM\SYSTEM\CurrentControlSet\Control\Terminal Server\WinStations\RDP-Tcp → PortNumber → cambia da 3389 a porta personalizzata. Aggiorna regola firewall di conseguenza.

Backup & Disaster Recovery

Il backup è la responsabilità più critica del sistemista. Un sistema di backup non testato non è un backup. La regola 3-2-1 è lo standard minimo del settore.

La Regola 3-2-1

3

Copie dei Dati

Mantieni almeno 3 copie dei dati: l'originale più 2 backup. La probabilità di perdita di tutte e 3 le copie è praticamente nulla.

2

Tipi di Supporto

Su almeno 2 supporti diversi (es. NAS locale + nastro, o disco + cloud). Guasti hardware tendono a colpire lo stesso tipo di supporto.

1

Copia Offsite

Almeno 1 copia fuori sede (offsite o cloud). Protegge da incendio, alluvione, furto e ransomware che colpisce la rete aziendale.

Tipi di Backup

Tipo Descrizione Tempo Backup Tempo Restore Spazio
Full Copia completa di tutti i dati Lungo Breve Alto
Incrementale Solo modifiche dall'ultimo backup (full o incr.) Breve Lungo (catena) Basso
Differenziale Solo modifiche dall'ultimo full Medio Medio Medio
Snapshot VM Punto di ripristino della VM intera Minuti Minuti Variabile
Replica Copia in tempo reale (es. DFS, vSAN) Continuo Immediato Alto (2x)

Sistema

Risposta a un Incidente di Sicurezza

  1. 1

    Contenimento immediato: Isola il sistema dalla rete (disconnetti dal vSwitch VMware, blocca nel firewall, spegni se necessario). Impedisci la propagazione laterale.

  2. 2

    Raccolta evidenze: Prima di spegnere, acquisisci: memoria RAM (dump), log di sistema, connessioni di rete attive, processi in esecuzione. Le evidenze volatili spariscono allo spegnimento.

  3. 3

    Analisi: Identifica il vettore di attacco (phishing, vuln, credenziali rubate), l'impatto (dati esfiltrati? ransomware? lateral movement?) e il periodo di compromissione.

  4. 4

    Eradicazione: Rimuovi malware, chiudi vettori di accesso, reset credenziali compromesse, patching vulnerabilità sfruttate.

  5. 5

    Ripristino: Restore da backup pulito (pre-compromissione), verifica integrità sistemi, ripristino graduale dei servizi con monitoraggio intensificato.

  6. 6

    Post-mortem: Documenta l'incidente, le azioni intraprese, i tempi di risposta. Identifica miglioramenti al processo e alle difese. Aggiorna playbook.

Ansible — Playbook per hardening server Linux
# hardening.yml
---
- name: Hardening base server Linux
  hosts: all_servers
  become: yes

  tasks:
    - name: Aggiorna tutti i pacchetti
      apt:
        upgrade: dist
        update_cache: yes

    - name: Installa fail2ban
      apt:
        name: fail2ban
        state: present

    - name: Configura SSH hardening
      lineinfile:
        path: /etc/ssh/sshd_config
        regexp: "{{ item.regexp }}"
        line: "{{ item.line }}"
      loop:
        - { regexp: '^PermitRootLogin', line: 'PermitRootLogin no' }
        - { regexp: '^PasswordAuthentication', line: 'PasswordAuthentication no' }
        - { regexp: '^Port', line: 'Port 2222' }
      notify: reload sshd

  handlers:
    - name: reload sshd
      service:
        name: sshd
        state: reloaded

# Esegui playbook su tutti i server
ansible-playbook -i inventory.yml hardening.yml --check  # dry run
ansible-playbook -i inventory.yml hardening.yml
